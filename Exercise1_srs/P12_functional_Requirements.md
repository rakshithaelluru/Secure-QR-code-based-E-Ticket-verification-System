# Functional Requirements
FR-01: Ticket Generation – The system shall generate a unique QR code for each ticket.

FR-02: Ticket Storage – The system shall store all ticket details securely in the database.

FR-03: QR Code Scanning – The system shall allow the verifier to scan QR codes using a camera device.

FR-04: Ticket Validation – The system shall verify the ticket by checking its status in the database.

FR-05: One-Time Ticket Usage – The system shall prevent a ticket from being used more than once.

FR-06: User Management – The system shall allow the admin to manage system users.

FR-07: Report Generation – The system shall generate reports on ticket verification and usage.
